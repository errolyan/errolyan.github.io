## 1. 对话式聊天机器人智能体
首先从简单的聊天机器人智能体入手，它能解答用户提出的问题。

例如，你可以设置 Slack、Telegram 或网页聊天触发器，将用户消息传入 AI 智能体节点（使用 OpenAI 的 GPT 或 Google Gemini 等大语言模型），然后返回响应结果。n8n 的 AI 智能体节点可配置为使用任意聊天模型，还能串联“记忆节点”（Memory node）来保存对话上下文。

- **触发器**：例如用于 Slack/Telegram 的“新消息”节点，或 Webhook 表单。
- **AI 智能体节点**：选择你使用的大语言模型（OpenAI、Gemini 等），设置系统/用户提示词，必要时启用记忆功能。还可选择性添加工具（如 Google 搜索/SerpAPI 节点）以获取实时信息。
- **返回响应**：将 AI 智能体的输出连接到消息节点（Slack、Telegram 或 HTTP 响应），将答案发送给用户。

这是 n8n 中最基础的 AI 聊天工作流——n8n 的官方文档中甚至展示了一个案例：通过 OpenAI + SerpAPI 搭配记忆缓存，实现“具备响应式对话能力”的问答功能。

在实际应用中，你可以接入任意消息界面。例如，有社区案例借助 n8n 和新版 DeepSeek 大语言模型，构建了支持长期记忆和 Google 文档集成的 Telegram 机器人。这表明，即使代码基础薄弱，你也能打造出有趣、友好的聊天机器人（甚至是 Slack 上的“数字同事”）。

该方案的优势在于：你能获得一个 24 小时在线的助手，可处理咨询或任务分流，非常适合用于常见问题解答（FAQs）、团队服务台，或作为有趣的“公司助手”机器人。

案例参考：n8n 官方提供的“AI 智能体聊天”模板及教程，清晰展示了这种聊天模式。（即使是新手，搭建聊天触发器和 AI 智能体节点的过程也十分简单。）


## 2. 社交媒体内容生成智能体
想自动生成 Twitter/X 或博客内容吗？

AI 内容生成智能体可抓取热点趋势并自动撰写内容。

例如，n8n 模板展示了如何获取 Google 热门搜索趋势，借助 AI 撰写由 5 条内容组成的 Twitter 线程，并自动发布。还有用户构建了“SEO 博客撰写”智能体：它从 n8n 表单中获取关键词，通过自定义搜索（Custom Search）和文本提取 API 抓取 Google 搜索结果首页内容，再将这些内容传入 GPT，撰写并“拟人化”处理一篇约 1500 字的文章。

### （示例：Twitter 线程）搭建步骤：
1. **定时/触发节点**：例如设置每日定时触发，或使用 Webhook 触发器。
2. **趋势来源**：通过 BigQuery/Google Trends 或 RSS/自定义搜索节点获取热门话题。
3. **AI 筛选（可选）**：将趋势传入 AI 智能体，筛选出最相关的话题。
4. **AI 内容创作**：将话题或关键词传入 AI 智能体节点（GPT-4o、Gemini 等），并添加提示词（如“撰写一篇有吸引力的 Twitter 线程”）。
5. **发布**：通过 Twitter（X）节点发布第一条推文，再通过循环发布回复，组成完整线程。

你可以通过提示词调整内容风格（如幽默风、营销风）。n8n 的社交媒体工作流也体现了这一点：有一个模板使用 GPT-4o“生成对话式线程”，并按顺序发布每条推文；另一个模板则每日查找热点，自动发布 5 条有洞察力的推文。正如 n8n 官方所说，这些工作流能“从多个来源收集内容，借助 AI 分析，安排最佳发布时间”，甚至还能跟踪内容表现。

该方案的优势在于：你只需投入少量精力，就能保持内容的时效性。无需再对着空白屏幕苦思冥想——AI 会为你完成内容撰写。例如，内容创作者可借助这类机器人，每日稳定分享实用技巧或新闻。

案例参考：n8n 博客指南详细介绍了如何通过热点自动生成 Twitter 线程；上文提到的 Reddit SEO 智能体（第 16 项）也是真实用户案例。这些案例都证明：借助 AI + n8n，可实现社交媒体/博客内容的自动化代笔。


## 3. AI 驱动的网页爬虫智能体
传统网页数据抓取往往依赖易失效的选择器（selectors），而 AI 智能体可简化这一过程。在 n8n 中，你能构建“基于视觉的爬虫智能体”，无需关注原始 HTML 结构。例如，某个工作流的逻辑是：接收用户查询，通过 ScrapeBee API 节点截取网页截图或获取页面内容，再将内容传入 AI 智能体（如 Google Gemini）提取信息。通过这种方式，智能体“无需担心 DOM 结构，即可自动提取数据”。

### （基于视觉）搭建步骤：
1. **触发器**：通过聊天或 Webhook 节点输入查询内容或 URL 列表。
2. **URL 列表（可选）**：从 Google 表格或数据库中提取 URL。
3. **爬虫 API**：使用 ScrapeBee（或其他渲染 API）捕获网页截图/文本。
4. **AI 智能体**：将抓取到的 HTML/图片传入 AI 智能体节点，通过提示词让其解析或总结内容。
5. **存储输出**：将结构化数据保存回表格/数据库，以备后续使用。

n8n 还支持结合 AI 的“传统爬虫”方案。

例如，ReAct AI 智能体可通过 HTTP 请求节点获取网页，再使用推理/操作工具解析页面。简而言之，该智能体“获取网页内容”后，通过 AI 逻辑处理 HTML。无论采用哪种方式，你都无需再费力编写 CSS/XPath 选择器。

该方案的优势在于：AI 的灵活性可应对结构混乱的网页。即使网站布局变更，AI 也能自适应调整，避免爬虫失效。

案例参考：n8n 官方工作流“基于视觉的 AI 爬虫”展示了该方案的实际应用。在企业场景中，该智能体可用于文章总结，或从格式不统一的网页来源中收集数据。


## 4. 自然语言数据查询与可视化智能体
该智能体可将自然语言问题转化为数据查询指令，并生成洞察结果。它能接收文本提示，为数据库生成 SQL 查询语句或图表。

例如，n8n 的“SQL 智能体”案例使用 OpenAI 和 quickchart.io API：智能体提取信息后，执行 Postgres 查询，必要时还能自动生成图表。


![](https://fastly.jsdelivr.net/gh/bucketio/img15@main/2025/10/05/1759631691175-971466fd-0b8e-4b54-9f6c-6db23482d731.png)

### 搭建步骤：
1. **触发器**：通过聊天表单或定时节点接收查询请求（如“按地区展示销售额”）。
2. **AI 智能体**：配置 AI 智能体节点，添加工具（如 MySQL/Postgres/SQLite 等 SQL 数据库节点）。在系统提示词中可明确指令：“生成 SQL 查询语句以解答该问题”。
3. **SQL 执行**：智能体将生成的 SQL 语句发送至数据库节点执行。
4. **可选分析**：若结果数据量较大，可将其传入另一个 AI 节点或文本分类器，总结关键发现。
5. **可选图表生成**：若智能体判断需用图表辅助展示，可通过 QuickChart 节点绘制数据图表。

在实际案例中，有指南将基于 Python 的分析聊天工具重构为 n8n 智能体：AI 节点配置了 SQL 执行工具和记忆功能，可“自动生成 SQL 查询语句，并发送至查询工具执行”。最终效果与原系统一致，但团队中任何人无需编码即可维护该智能体。n8n 官方的 SQL 智能体模板（搭配 OpenAI）也具备类似功能：当 AI 认为有必要时，会自动生成图表。

该方案的优势在于：你无需手动编写 SQL，即可通过自然语言查询数据。企业创始人可借助它快速生成仪表盘，或解答临时数据查询需求。

案例参考：某案例研究中构建了“供应链控制塔”系统，支持“聊天转 SQL”功能——顾问可通过自然语言提问，直接获取数据结果。n8n 教程还展示了如何为查询结果自动添加图表。


## 5. 电子表格/Airtable 对话式智能体
该智能体可为非技术团队提供“聊天式数据访问界面”。例如，将 Google 表格或 Airtable 转化为基于聊天的 AI 知识库。

n8n 的“AI 数据分析师”模板会将表格行数据加载到 NocoDB 中，以便大语言模型通过自然语言解答相关问题。

类似地，另一个工作流构建了 Airtable 专属 AI 聊天工具：你用自然语言提问后，智能体通过 Airtable API 节点查询数据，并返回结果。


![](https://fastly.jsdelivr.net/gh/bucketio/img15@main/2025/10/05/1759631707818-b405c5a6-562a-458b-a457-60aff02f103d.png)


### 搭建步骤：
1. **数据设置**：将 Google 表格/Airtable 接入 n8n。对于表格，可同步至数据库或 NocoDB；对于 Airtable，通过 Airtable 节点查询记录。
2. **触发器**：通过聊天表单或消息节点接收用户问题。
3. **AI 智能体**：在 AI 智能体节点中嵌入相关数据集（通过记忆功能或工具）。例如，提供 JSON 模式链接，或使用 n8n 的数据存储（Data Stores）。
4. **查询与解答**：AI 智能体解析问题后，从表格中获取数据，并格式化答案。
5. **回复**：通过聊天或邮件将 AI 生成的答案发送给用户。

该方案的优势在于：非技术用户无需在表格中逐一查找，即可快速获取数据洞察。

案例参考：n8n 模板将表格数据存储为可搜索格式，支持通过聊天查询；“Airtable 聊天工具”则展示了如何通过对话查询数据库。这些方案已被验证能有效降低数据访问门槛。


## 6. 邮件总结智能体
该智能体可将杂乱的收件箱转化为简洁的每日摘要。它能一键获取邮件并生成总结，在 n8n 中的实现逻辑如下：设置定时触发器（如早上 8 点），连接 Gmail 或 IMAP 节点获取最新邮件，将邮件内容汇总后传入 AI 智能体或“总结”（Summarize）节点，最后通过邮件或 Slack 发送关键信息。

### 搭建步骤：
1. **定时触发器**：例如“每天早上 8 点”触发。
2. **邮件获取**：通过 Gmail/IMAP 节点从收件箱（或特定标签）中提取未读邮件。
3. **AI 总结**：将所有新邮件内容传入 AI 智能体，或使用专用 OpenAI 提示词（如“总结这些邮件并列出待办事项”）。
4. **分发总结**：通过邮件（SMTP/发送节点）或 Slack/Teams 频道发送 AI 生成的总结。
5. **可选标记已读**：通过邮件节点标记或归档已处理邮件，避免重复处理。

这是经典的效率提升工具。n8n 官方工作流可获取 Gmail 邮件，总结关键信息，并在早晚各发送一次简短更新。你无需再面对收件箱疲劳——无需阅读数十封完整邮件，只需查看一份简洁报告即可。

该方案的优势在于：每周可帮你节省数小时时间。研究表明，人们在邮件处理上浪费大量时间，而 AI 可完成初步筛选阅读。

## 7. 会议转录与总结智能体
有了该智能体，你再也不会遗漏会议细节。它能接收 Zoom/Teams 会议录制文件，生成笔记或待办事项。在 n8n 中，可通过云存储节点或 Recall.ai 集成触发工作流（当新录制文件生成时），将转录文本传入 AI 进行总结，最后通过邮件发送或保存总结结果。

### 搭建步骤：
1. **会议录制获取**：连接节点接收录制文件或转录文本（如从 Zoom 上传，或使用 Rev/Recall.ai 等 API）。
2. **转录文本存储**：若有需要，将原始文本保存到数据库或云存储。
3. **AI 总结**：将会议文本传入 AI 智能体，通过提示词（如“突出关键决策和待办事项”）生成总结。
4. **输出**：将总结通过邮件发送给参会者，或同步至 Slack/Notion。

该方案的优势在于：自动生成会议纪要能大幅提升效率。你无需再匆忙记笔记，AI 会生成可靠的总结内容。

案例参考：n8n 博客工作流使用 Recall.ai 进行转录，通过 Supabase/Postgres 存储文本，再借助 OpenAI 实时生成总结。该方案的创建者表示，它比现成的会议工具更灵活——你可自定义笔记的捕获和存储方式。


## 8. AI 客户支持智能体
该智能体可加快支持工单处理速度，能对客户咨询进行分流并自动撰写回复。例如，在 n8n 中设置 Zendesk 或邮件触发器。

它会调用内部知识库（Google Drive 文档、Git 上的 Markdown 文件等）生成贴合上下文的回复；若回复可信度较低，则会将工单分配给人工处理。

### 搭建步骤：
1. **触发器**：例如 Zendesk/Help Scout 中的“新工单”，或 Gmail 收到的新支持邮件。
2. **分配检查（可选）**：通过开关节点跳过已分配给人工的工单。
3. **AI 分类**：将客户问题传入 AI 智能体，对问题类型进行分类（如账单问题、技术问题等）。可通过条件节点自动处理已知类型的问题。
4. **知识库查询**：对于未分类问题，查询知识库：例如调用 Google Drive（或其他节点）获取相关文档，或使用搭载向量数据库（存储常见问题）的 AI 智能体。
5. **AI 撰写回复**：智能体根据知识库内容撰写回复，可选添加“可信度检查”（通过另一个提示词），确保回复无不确定表述。
6. **回复**：若回复符合要求，通过 Zendesk 节点发布草稿回复（或发送邮件）；否则通知人工介入。

这类方案并非空想——已有成熟模板可用。例如，某个 n8n 工作流可端到端自动化工单处理：对新工单分类、查询 Mintlify 文档，甚至还能检查回复的风格和语气。正如 n8n 官方所说，该工作流“借助 AI 驱动的分类、回复生成和安全检查，仅基于官方文档即可自动化支持工单处理”。换句话说，无需人工逐句撰写，它就能为客户提供准确回复。

该方案的优势在于：可缓解工单积压问题。即使有 100 个工单进入系统，若 AI 能可靠处理其中 70%，支持成本也会大幅降低。（如今已有许多企业使用 AI 辅助撰写回复。）

案例参考：上文提到的“Mintlify & Zendesk AI”模板是功能完整的支持机器人（涵盖邮件、AI、审批流程）。案例研究显示，它能将响应时间缩短 80%——快速处理大量工单，让人工支持人员聚焦复杂问题。


## 9. 知识库（RAG）查询智能体
该智能体可扩大内部知识的覆盖范围。大型企业通常拥有大量文档，而 AI 知识库智能体可充当“内部谷歌”，其核心是“检索增强生成”（RAG）技术：员工提问后，智能体搜索企业文档（PDF、Drive 文件等），并基于最新信息给出答案。

### 搭建步骤：
1. **数据摄入**：通过 Google Drive 节点或 Webhook 监控新增/更新的文档。当有文件新增时，下载文件并通过嵌入节点（如 OpenAI 嵌入）将其传入向量数据库（如 Pinecone）。
2. **查询触发**：员工通过聊天机器人或表单输入问题。
3. **搭载工具的 AI 智能体**：为 AI 智能体配置“向量搜索”工具。在提示词中明确指令：让智能体检索相关文档片段（工具可查询 Pinecone）。
4. **生成答案**：智能体基于检索到的文本，撰写自然语言答案。
5. **回复**：在聊天界面（Slack、微软 Teams 或网页界面）中返回答案。

图：AI 智能体查询 Google Drive 中的企业文档流程——将文档摄入 Pinecone 后，通过聊天与向量数据库交互。

该方案的优势在于：员工无需再花费大量时间查找信息。RAG 机器人能即时找到答案，无需再“询问同事 X”或阅读过时手册。

案例参考：n8n 的“企业文档 AI 智能体”模板正是采用这种方案：基于 Google Drive 文件构建 RAG 聊天机器人。如上图所示，该工作流会持续为新文档建立索引，员工无需浏览文件夹，通过自然语言即可提问。用户反馈称，它能快速定位隐藏在文档中的政策或技术信息，每周可节省数小时。


## 10. 安全警报增强智能体
主动网络安全是该智能体的应用场景。它能自动增强安全警报信息并进行分流。例如，若 SIEM（安全信息和事件管理）系统发送警报（如通过 Zendesk 工单），智能体可通过大语言模型提示词为警报添加 MITRE ATT&CK 战术标签、划分紧急程度，并提供补救建议。

### 搭建步骤：
1. **触发器**：新安全警报传入（通过 Webhook、Zendesk 或日志系统节点）。
2. **AI 信息增强**：将警报详情传入 AI 智能体，通过提示词（如“根据警报详情，映射对应的 MITRE ATT&CK 技术，并推荐应对措施”）生成增强信息。
3. **标签添加**：通过另一个节点或智能体输出，为警报添加标签/备注（如在工单中添加 MITRE 标签、高/中优先级标识）。
4. **补救指导**：智能体可附加建议步骤（如“建议：拦截该 IP 地址，修复该 CVE 漏洞”）。
5. **通知团队**：将增强后的警报发送至 Slack/邮件，供安全运营团队处理。

该方案的优势在于：安全团队通常任务繁重，AI 可标准化并加快警报分流流程。

案例参考：2025 年，n8n 演示了“SIEM 警报增强”工作流：它能自动为警报添加 MITRE 技术标签，甚至在 Zendesk 工单中填充威胁情报。这意味着，AI“初级分析师”可即时完成原本人工需 5-10 分钟/个的警报处理工作。


## 11. API/文档查询智能体
该智能体可帮助开发者或用户查询技术文档。例如，API 文档聊天机器人能通过获取官方文档解答问题。n8n 展示过一个 GitHub API 案例：当你提问“如何列出仓库？”时，工作流会通过 HTTP 获取最新的 GitHub API 文档，再将文档传入大语言模型生成答案。

### 搭建步骤：
1. **触发器**：通过聊天机器人界面或快捷指令（如 Slack 中的指令）触发。
2. **HTTP 获取**：AI 智能体通过 HTTP 请求节点（或内置工具）获取最新文档页面（如原始 Markdown/HTML）。
3. **AI 生成答案**：智能体读取文档内容，生成实用解答。
4. **返回答案**：在聊天或界面中返回答案。

该方案的优势在于：API 文档会不断更新，而大语言模型的知识可能过时——该智能体可获取实时信息，让开发者直接从官方来源快速获取答案。

案例参考：n8n 的“GitHub API 文档智能体”展示了该方案的应用：AI 智能体“支持与 GitHub API 文档对话”，通过工作流传入文档 URL，无需离开聊天窗口，即可获取最新的 API 使用指南。


## 12. 模型微调智能体
该智能体可根据你的数据定制 AI 模型。一个高阶方案是构建 AI 模型微调智能体：例如，你可将问答对或领域特定文本上传至 Google Drive，然后让 n8n 自动基于这些数据微调 OpenAI 模型。

### 搭建步骤：
1. **触发器**：例如，新训练文件（JSONL 格式）传入 Drive/Dropbox 文件夹（通过文件监控或 Webhook 节点触发）。
2. **格式检查（可选）**：确认文件格式符合 OpenAI 微调规范。
3. **OpenAI 微调节点**：通过 OpenAI 节点，使用该文件创建微调任务。
4. **监控**：通过循环调用 OpenAI“列出微调任务”API，检查任务状态。
5. **更新**：微调完成后，存储新模型的名称/密钥，以备后续使用。

该方案的优势在于：无需编码，即可将数据转化为定制化 GPT 模型。

案例参考：n8n 提供的工作流可在几分钟内，基于 Google Drive 文档微调 OpenAI 模型（官方提醒需注意隐私数据安全）。此外，对于本地部署场景，你可在 n8n 中连接 Ollama 或本地大语言模型，使用自有向量训练模型。无论采用哪种方式，该智能体都能增强 AI 在特定任务中的表现。
